VacationSavers APIs Documentation (1.0.0)

First of All

Every process of VacationSavers API, TokenAuth is required. Please contact VacationSavers to obtain organization account and TokenAuth.

Single Signon Process

First time customer need to submit their information to create customer account under organization entity.
To create account, CreateUser process is needed, you can see the technical detail for required fields and response from the link at the bottom.
You can also try to submit and see the response from Try It Out link too.

Here are the basic data required to create account.
  • "username": "string" - user name will be key for other processes and must be unique, so we highly recommend to put organization entity as profix. VACATIONSAVERS_user1, for example.
  • "email": "string"
  • "first_name": "string"
  • "last_name": "string"
  • "address": "string"
  • "city": "string"
  • "state_code": "string"
  • "country_code": "string"
  • "postal_code": "string"
  • "phone": "string"

If the process is done successfully, 20 digits token TokenUsr will be returned. Then you can use TokenUsr to access VacationSavers using the below format.
https://www.vacationsavers.com?auth_token=TokenUsr

Supplemental Processes

The additional processes those may help

  • GetAuthToken : Get TokenUsr for the submitted username.
  • DeactivateUser : Pause or temporaly disable the submitted username.
  • ReActivateUser : Resume the submitted username.
  • GetUserList : Get all customer accounts under the organization account.
